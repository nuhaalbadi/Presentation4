› Cognitive Walkthroughs

The cognitive walkthrough is a usability inspection method that does not require users and can be done in early UI development stages. The cognitive walkthrough requires having a set of tasks that the UI support, description of the UI or mockup, and intended users background. Once all these three elements are available, UI evaluators/designers “walkthrough” each task while reasoning whether or not each step required to perform the task can be easily recognized and done by the user.

The cognitive walkthrough was developed by Clayton Lewis, Peter Polson, Cathleen Wharton, and John Rieman in 1992. If you would like to learn about the theory behind the method, you can do that by reading Cognitive Walkthroughs: A Method for Theory-Based Evaluation of User Interfaces by Ploson et al.
